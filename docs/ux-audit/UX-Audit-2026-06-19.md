# Stock Analyze AI Connect — UX / Design Audit

Audit date: 2026-06-18–19  
Target: authenticated production site at `stock-analyze-ai-connect.vercel.app`  
Viewports: desktop 1440×900; mobile 390×844  
Scope: information hierarchy, contrast/readability, spacing consistency, responsive behavior, interaction feedback, accessibility, and overlapping mobile/technical SEO checks.

## Executive summary

- Critical: 0
- High: 3
- Medium: 5
- Low: 3
- Safe fixes applied: 9 groups
- Screenshot coverage: login plus all 12 authenticated routes on desktop; login plus 11 routes on mobile. The Auto Trading mobile DOM was audited at 390 px, but its image capture repeatedly timed out.

The product has a coherent dark terminal aesthetic and generally stable responsive shell. The biggest systemic risks are very small text, incomplete form/control semantics, and dense nested navigation in Auto Trading. These are more consequential than cosmetic polish because they affect comprehension, keyboard/screen-reader use, and mobile usability.

## Findings by severity

### High

#### H1 — Dense 9–10 px text is below a comfortable mobile reading size

Small uppercase labels are used as primary information across the shell and Auto Trading. On mobile this makes configuration and status content strenuous to scan, especially when paired with muted opacity.

- Evidence: [Auto Trading desktop](screenshots/2026-06-18/ux-autotrading-desktop.png), [Backtest mobile](screenshots/2026-06-18/ux-backtest-mobile.png)
- Code: `src/components/AutoTrading/StrategyFlowBuilder.tsx:130`, `src/components/AutoTrading/StrategyFlowBuilder.tsx:183`, `src/components/AutoTrading/RiskControlPanel.tsx:290`, `src/terminal/shell/TopNav.tsx:313`
- Recommendation: establish 12 px as the minimum for functional labels and 14–16 px for body text; reserve 9–10 px for nonessential metadata only.
- Status: partially fixed where contrast was objectively failing in Backtest and across Strategy Flow / Risk Control, where functional text is now at least 11 px. A global typography-scale change still requires a design-system decision.

#### H2 — Several inputs and clickable rows still lack robust semantics

Visible labels are often not programmatically associated with inputs, and some rows/list items rely on `onClick` without keyboard semantics. This creates screen-reader ambiguity and keyboard dead ends.

- Inputs: `src/components/backtest/BacktestHeaderSection.tsx:147`, `src/components/AutoTrading/RiskControlPanel.tsx:290`, `src/terminal/ui/SmartMoneyAlertSettingsPanel.tsx:353`, `src/terminal/pages/Research.tsx:206`
- Click targets: `src/terminal/pages/Dashboard.tsx:198`, `src/terminal/pages/Market.tsx:87`, `src/terminal/pages/News.tsx:134`, `src/terminal/pages/Crypto.tsx:58`
- Recommendation: connect every label with `htmlFor/id` or an enclosing label; render navigation targets as links/buttons, or add keyboard handling only when native semantics are impossible.
- Status: implemented in the follow-up pass for the audited Backtest, Risk Control, Smart Money, Research, Dashboard, Market, News, and Crypto surfaces. Additional forms outside this audit should adopt the same pattern when touched.

#### H3 — Auto Trading has competing navigation models and excessive cognitive load

The page exposes top-level Live/Strategy/Backtest/Simulation tabs, then another eight-tab control surface containing Strategy and Backtest again. Users cannot easily predict which layer owns a task.

- Evidence: [Auto Trading desktop](screenshots/2026-06-18/ux-autotrading-desktop.png)
- Code: `src/terminal/pages/AutoTrading.tsx:156`, `src/components/AutoTrading/AgentControlPanel.tsx:181`
- Recommendation: choose one primary task hierarchy. A sensible model is **Monitor / Stock Trading Strategies / Backtest / Broker & Risk**, with secondary tools inside each section.
- Status: recommendation only. This is an information-architecture decision.

### Medium

#### M1 — Market sector names overlapped at the mobile breakpoint

Two-column cards allowed long fund names to escape their cell, visually colliding with adjacent cards.

- Evidence: [Market mobile, before fix](screenshots/2026-06-18/ux-market-mobile.png)
- Code/fix: `src/terminal/pages/Market.tsx:35`, `src/terminal/pages/Market.tsx:52`
- Status: fixed with intrinsic-width constraints, truncation, and native button semantics.

#### M2 — Crypto table hides the rightmost trend without an affordance

At 390 px the five-column table compresses or clips the trend area, but there is no clear horizontal-scroll container or cue.

- Evidence: [Crypto mobile](screenshots/2026-06-18/ux-crypto-mobile.png)
- Code: `src/terminal/pages/Crypto.tsx:52`
- Recommendation: prioritize Pair / Price / 24h on mobile, move Trend into a disclosure, or add an explicit scroll container with a fade/label.
- Status: safely mitigated with an explicit horizontal-scroll container and stable 720 px table width. Column prioritization remains a product decision.

#### M3 — Mobile bottom navigation cannot represent the active location on seven routes

Only Dashboard, Market, Screener, Portfolio, and Alerts are present. On Research, Smart Money, Backtest, Auto Trading, News, Crypto, and Settings, none of the five items is active.

- Code: `src/terminal/shell/Layout.tsx:43`, `src/terminal/shell/Layout.tsx:160`
- Recommendation: add a “More” destination that becomes active for secondary routes, or make the fifth item contextual.
- Status: recommendation only.

#### M4 — Touch targets are frequently smaller than 44–48 px

Several icon controls use roughly 24–32 px hit areas. The new accessible names improve assistive technology output but not motor accessibility.

- Code: `src/components/AutoTrading/StrategyTab.tsx:38`, `src/terminal/ui/SmartMoneyAlertSettingsPanel.tsx:272`, `src/terminal/pages/Dashboard.tsx:137`
- Recommendation: standardize an icon-button component with at least 44×44 px interactive area while keeping the glyph visually compact.
- Status: fixed for the audited high-frequency Strategy view, Smart Money, and Dashboard icon actions. A reusable global icon-button primitive remains recommended.

#### M5 — Heading hierarchy is inconsistent in complex views

Portfolio and News previously lacked a stable page-level heading; Backtest still jumps from `h1` to `h3` before `h2`.

- Code/fix: `src/terminal/pages/Portfolio.tsx:89`, `src/terminal/pages/News.tsx:44`
- Remaining: `src/components/backtest/BacktestHeaderSection.tsx:81`, `src/components/backtest/BacktestHeaderSection.tsx:142`, `src/components/backtest/BacktestHeaderSection.tsx:252`
- Status: fixed. Backtest now follows `h1` → section `h2` → strategy-parameter `h3`; Portfolio and News retain their added page headings.

### Low

#### L1 — User-facing copy exposed an internal filename

Smart Money described the feature as a “Follow.md workflow,” which reads like implementation leakage.

- Code/fix: `public/locales/en/translation.json:316`, `public/locales/zh/translation.json:317`
- Status: fixed.

#### L2 — “Data WorkFlow” does not exist in the deployed UI or source

The exact phrase was not found. The nearest matching concept is the strategy configuration heading in Auto Trading. It has been renamed as requested:

- English: **Stock Trading Strategies**
- Chinese: **股票交易策略**
- Code/fix: `public/locales/en/translation.json:752`, `public/locales/zh/translation.json:753`

#### L3 — Primary action colors vary by feature

Amber, cyan, violet, emerald, and rose are all used as prominent actions. The palette suits a terminal product, but action meaning is not fully consistent.

- Evidence: [Alerts desktop](screenshots/2026-06-18/ux-alerts-desktop.png), [Auto Trading desktop](screenshots/2026-06-18/ux-autotrading-desktop.png), [Smart Money desktop](screenshots/2026-06-18/ux-smartmoney-desktop.png)
- Recommendation: define semantic roles for primary, informational, success, warning, and destructive actions before changing colors.
- Status: recommendation only.
